The four customization layers

Higher number = wins on conflict · all layers merge & are sent to the model

1. Personal
github.com/copilot → Personal instructions
2. Repository
.github/copilot-instructions.md
+ path-scoped .github/instructions/*
3. Agent files
AGENTS.md / CLAUDE.md / GEMINI.md
4. Org / Enterprise
Org settings → Copilot → Custom instructions (Business / Enterprise)

Plus a user-profile layer: VS Code ~/.copilot/instructions/ · CLI ~/.copilot/copilot-instructions.md.

~10 minutes on customization. All layers merge — nothing is dropped; numbering (1 > 2 > 3 > 4) only matters on direct conflict. Path-specific uses applyTo: glob frontmatter — powerful for monorepos. Conflicting rules across layers are the #1 beginner pitfall. This is the single slide to photograph.

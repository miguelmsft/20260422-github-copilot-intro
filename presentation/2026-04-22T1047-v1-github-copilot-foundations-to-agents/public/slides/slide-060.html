How Copilot picks a skill

Session start
Scan skill sources
.github/skills/ · ~/.copilot/skills/ · plugins · COPILOT_SKILLS_DIRS · built-ins
Build catalog of (name, description) pairs
only metadata loaded
You prompt: "debug the failing CI on PR #42"
Model matches description
selects github-actions-failure-debugging
Built-in skill tool injects SKILL.md body into context
Copilot follows the steps
may run bundled scripts
Only descriptions load at session start; the body is injected on demand. That's how you can have dozens of skills without blowing context.

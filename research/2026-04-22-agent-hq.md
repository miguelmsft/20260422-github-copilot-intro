## Executive Summary

**Agent HQ** is GitHub's platform-level strategy, announced by COO Kyle Daigle at **GitHub Universe 2025 on October 28, 2025** ([Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)), for making AI coding agents native to the GitHub workflow. Rather than shipping a single "best" agent, GitHub is turning GitHub.com itself into an **open ecosystem** where developers can orchestrate multiple agents — GitHub Copilot plus partner agents from **Anthropic (Claude), OpenAI (Codex), Google, Cognition, and xAI** ([announcement post](https://github.blog/news-insights/company-news/welcome-home-agents/)) — from one place, while keeping work attached to the primitives developers already trust (Git, issues, pull requests, Actions, and the existing paid Copilot subscription).

The core user-facing surface is **"mission control"** — a unified command center, available across GitHub.com, VS Code, GitHub Mobile, and the Copilot CLI, used to assign tasks to agents, run them in parallel, watch session logs, steer them mid-run, and review their pull requests ([announcement post](https://github.blog/news-insights/company-news/welcome-home-agents/); [mission-control how-to](https://github.blog/ai-and-ml/github-copilot/how-to-orchestrate-agents-using-mission-control/)). Enterprise features layer on top: an **AI Controls** control plane for policy/audit, **Copilot Code Quality** and **automated first-pass code review**, a **Copilot metrics dashboard**, and AGENTS.md-based **custom agents** in both VS Code and Copilot CLI ([announcement post](https://github.blog/news-insights/company-news/welcome-home-agents/); [Agent management for enterprises — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/enterprise-management)).

As of the reporting date (April 2026), Agent HQ is being **rolled out incrementally** — it is not a single shippable SKU. Mission control has shipped ([Dec 1, 2025 how-to post](https://github.blog/ai-and-ml/github-copilot/how-to-orchestrate-agents-using-mission-control/)); **Claude (Anthropic) and OpenAI Codex went into public preview on GitHub.com, GitHub Mobile, and VS Code 1.109+ on February 4, 2026**, limited to **Copilot Pro+ and Copilot Enterprise** subscriptions ([Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/); [changelog entry](https://github.blog/changelog/2026-02-04-claude-and-codex-are-now-available-in-public-preview-on-github)). Google, Cognition, and xAI integrations were announced but not yet released at the time of this research ([Pick your agent](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)). Each agent session consumes one premium request, agents "can still make mistakes," and output is designed to be reviewed ([Pick your agent](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)) — important caveats to communicate to beginner audiences.

## 1. Overview

### What It Is

Agent HQ is **GitHub's vision and platform layer for orchestrating multiple AI coding agents inside GitHub**. It is not a standalone product you buy separately — it is a set of capabilities rolled out across the existing Copilot subscription that let you pick *which* agent runs a task, run several in parallel, and manage them with GitHub's native workflow primitives.

> "At GitHub Universe, we're announcing Agent HQ, GitHub's vision for the next evolution of our platform. Agents shouldn't be bolted on. They should work the way you already work. That's why we're making agents native to the GitHub flow. Agent HQ transforms GitHub into an open ecosystem that unites every agent on a single platform."
> — Source: [Introducing Agent HQ: Any agent, any way you work — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/) (Kyle Daigle, Oct 28, 2025)

### Why It Matters (Problem It Solves)

GitHub frames Agent HQ as the answer to **tool fragmentation** in the AI coding space — developers juggling separate dashboards, subscriptions, and prompt windows for each agent vendor.

> "The current AI landscape presents a challenge we're all too familiar with: incredible power fragmented across different tools and interfaces."
> — Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)

> "Context switching equals friction in software development. Today, we're removing some of that friction with the latest updates to Agent HQ which lets you run coding agents from multiple providers directly inside GitHub and your editor, keeping context, history, and review attached to your work."
> — Source: [Pick your agent: Use Claude and Codex on Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/) (Mario Rodriguez, Feb 4, 2026)

### Key Features (Announced at Universe 2025)

From the Universe 2025 announcement post, Agent HQ bundles:

- **Mission control** — a single command center (on github.com, VS Code, GitHub Mobile, Copilot CLI) to assign, steer, and track multiple agents.
- **Plan Mode in VS Code** — Copilot asks clarifying questions and builds a step-by-step plan before any code is written.
- **Custom agents via AGENTS.md** — source-controlled files that give agents persona, rules, and guardrails.
- **GitHub MCP Registry** — one-click discovery/install of MCP servers (Stripe, Figma, Sentry, etc.) in VS Code.
- **Agentic code review** — a first-pass Copilot review step built into the coding agent workflow before a human sees the PR.
- **GitHub Code Quality (public preview)** — org-wide maintainability/reliability checks beyond security.
- **Copilot metrics dashboard (public preview)** — org-wide visibility into Copilot/agent usage and impact.
- **AI Controls / control plane** — enterprise admin console to govern which agents and models are allowed, with audit logging.
- **New branch controls & identity features** — granular CI gating and per-agent identity, plus one-click merge-conflict resolution.
- **New integrations** — Slack and Linear (on top of earlier Jira, Microsoft Teams, Azure Boards, Raycast).

Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/).

## 2. Key Concepts

### 2.1 "Mission control" — the command center

Mission control is the user-facing experience of Agent HQ. It is **not a single website** but the same interaction model surfaced in four places.

> "This starts with a mission control, a single command center to assign, steer, and track the work of multiple agents from anywhere."
> — Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/) | Provenance: verbatim

> "It's not a single destination; it's a consistent interface across GitHub, VS Code, mobile, and the CLI that lets you direct, monitor, and manage every AI-driven task."
> — Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/) | Provenance: verbatim

In practice, mission control lets you:

- Assign a task to one agent — or the **same** task to several agents to compare results.
- Run tasks **asynchronously in parallel** across one repo or many.
- Watch **real-time session logs** showing the agent's reasoning, not just its actions.
- **Steer mid-run** — pause, refine, or restart.
- Jump directly into the resulting draft pull request.

> "We recently shipped Agent HQ's mission control, a unified interface for managing GitHub Copilot coding agent tasks. Now, you can now assign tasks to Copilot across repos, pick a custom agent, watch real-time session logs, steer mid-run (pause, refine, or restart), and jump straight into the resulting pull requests—all in one place."
> — Source: [How to orchestrate agents using mission control — The GitHub Blog](https://github.blog/ai-and-ml/github-copilot/how-to-orchestrate-agents-using-mission-control/) (Dec 1, 2025)

### 2.2 Agents stay on GitHub primitives

A deliberate design choice: Agent HQ does **not** introduce a new artifact type. Agent output shows up as draft pull requests, issue comments, and commits — the same things you already review.

> "Agent-generated changes show up as draft pull requests and comments, reviewed the same way you'd review a teammate's work. There are no new dashboards to learn, and no separate AI workflows to manage."
> — Source: [Pick your agent: Use Claude and Codex on Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

### 2.3 From sequential to parallel orchestration

The mental model GitHub pushes is: you stop babysitting one agent and become a **conductor** of several.

> "If you're already used to working with an agent one at a time, you know it's inherently sequential. … Mission control changes this. You can kick off multiple tasks in minutes—across one repo or many. … Instead of each task taking 30 seconds to a few minutes to complete, your agents might spend a few minutes to an hour on a draft. But you're no longer just waiting. You're orchestrating."
> — Source: [How to orchestrate agents using mission control — The GitHub Blog](https://github.blog/ai-and-ml/github-copilot/how-to-orchestrate-agents-using-mission-control/)

### 2.4 Architecture at a glance

```

                          │  (GitHub.com · VS Code · GitHub Mobile · CLI)   │
                          └───┬──────────────────┬───────────────────┬─────┘
                              │ assign / steer   │ logs / review     │ governance
                              ▼                  ▼                   ▼
        ┌──────────────┐  ┌──────────────┐  ┌────────────┐   ┌───────────────────┐
        │ GitHub       │  │ Claude       │  │ OpenAI     │   │ Google / Cognition│
        │ Copilot      │  │ (Anthropic)  │  │ Codex      │   │ / xAI (announced) │
        │ cloud agent  │  │              │  │            │   │                   │
        └─────┬────────┘  └──────┬───────┘  └─────┬──────┘   └─────────┬─────────┘
              └──────────┬───────┴─────────────────┴───────────────────┘
                         ▼
          GitHub primitives: repos · issues · pull requests · Actions · Checks
                         ▼
```

## 3. Getting Started (Beginner Path)

### Prerequisites

- A **paid GitHub Copilot** subscription. For third-party agents (Claude, Codex), specifically **Copilot Pro+** or **Copilot Enterprise**.
- For GitHub.com / Mobile: an account with Copilot enabled.
- For VS Code: **VS Code 1.109 or later**.
- Enterprise users: admin must **allow the third-party agent** via AI Controls.

Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/).

### Starting an agent session on GitHub.com

> "You can start and manage agent sessions directly from GitHub:
> - Open the Agents tab in any repository where agents are enabled.
> - Enter your request and select an agent using the Copilot icon.
> - Submit the request to start the session."
> — Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

### Assigning agents to issues / PRs

> "Assign an issue to Copilot, Claude, Codex, or all three to compare results. Agents can submit draft pull requests for review. Assign agents to existing pull requests to request changes or further analysis. Mention @Copilot, @Claude, or @Codex in pull request comments to prompt follow-up work."
> — Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

### Starting from VS Code

> "You can also work with agents directly from VS Code (version 1.109 or later):
> - Open the Agent sessions view from the title bar or by pressing Ctrl+Shift+P / CMD+Shift+P and searching for 'agent sessions'
> - Choose a session type and your agent: Local (Copilot), Claude, or Codex for fast, interactive assistance; Cloud for autonomous tasks that run on GitHub, then select an agent; Background for asynchronous local work (Copilot only)"
> — Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

### Billing

> "Each coding agent session consumes one premium request."
> — Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

## 5. Relationship to GitHub Copilot

**Agent HQ is delivered through the existing paid Copilot subscription** — there is no separate "Agent HQ" SKU, login, or billing surface.

> "You're accessing agents through your existing paid Copilot subscription."
> — Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)

Key points for the presentation:

1. **Copilot is the "first-party" agent** inside Agent HQ — the GitHub Copilot cloud agent, Copilot CLI, and Copilot agent mode in the IDE all plug into mission control. The GitHub Docs IA now organizes them under `Copilot / Concepts / Agents / Cloud agent`, `Copilot CLI`, `Third-party agents` (OpenAI Codex, Anthropic Claude), and `Enterprise management`. See [Agent management for enterprises — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/enterprise-management) and [About GitHub Copilot cloud agent — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-cloud-agent).
2. **Copilot CLI and VS Code agent sessions feed into mission control** — they are *surfaces* of the same Agent HQ orchestration model, not parallel products. The VS Code Agent sessions view explicitly lets users choose "Local (Copilot), Claude, or Codex" or a "Cloud" session that runs on GitHub, confirming the shared orchestration layer ([Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)). Mission control is also described as "a consistent interface across GitHub, VS Code, mobile, and the CLI" ([Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)).
3. **Agent HQ is a superset of "Copilot agent"** in the sense that it adds third-party agents and cross-agent orchestration on top of the Copilot coding agent — the announcement post positions Agent HQ as transforming "GitHub into an open ecosystem that unites every agent on a single platform" ([Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)).
4. **Everything is gated by Copilot policies and AI Controls** — so org admins already familiar with Copilot governance extend the same controls to third-party agents. Enterprise admins "define which agents and models are permitted across the organization" ([Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)), and the AI Controls view manages "enterprise-level availability of agents like Copilot cloud agent, Copilot code review, and Copilot custom agents" ([Agent management for enterprises — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/enterprise-management)).

## 6. Enterprise Features & Governance

Agent HQ's enterprise layer is organized around the **AI Controls** view in GitHub Enterprise.

> "The AI Controls view provides a centralized platform where you can manage and monitor AI policies and agents across your enterprise. From the 'Agents' page, you can:
> - Manage the enterprise-level availability of agents like Copilot cloud agent, Copilot code review, and Copilot custom agents
> - Control who can manage your enterprise-level custom agents
> - View and filter a list of agent sessions in your enterprise over the last 24 hours
> - Find a detailed record of agentic audit log events"
> — Source: [Agent management for enterprises — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/enterprise-management)

Policy granularity highlights from the same doc:

- Four policy states for Copilot cloud agent (including "Enabled for selected organizations," which can be scoped by organization **custom properties** — e.g., region, compliance tier, department).
- **Agent Mode in IDE chat** can be toggled independently from general Chat in IDE.
- **Custom agents** can be defined at enterprise, organization, or repository level.
- Enterprise management supports **REST API endpoints** for custom agents and cloud-agent management.

Additional enterprise-facing capabilities announced at Universe:

> "For enterprise administrators who are managing AI access, including AI agents and MCP, we're focused on providing consistent AI controls for teams with the control plane — your agent governance layer. Set security policies, audit logging, and manage access all in one place. Enterprise admins can also control which agents are allowed, define access to models, and obtain metrics about the Copilot usage in your organization."
> — Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/)

> "Agent HQ gives you org-wide visibility and systematic control over how AI interacts with your codebase:
> - Agent controls: Manage access and security policies in one place, allowing enterprise admins to define which agents and models are permitted across the organization.
> - Code quality checks: GitHub Code Quality (in public preview) extends Copilot's security checks to evaluate the maintainability and reliability impact of changed code…
> - Automated first-pass review: We have integrated a code review step directly into the Copilot's workflow, allowing Copilot to address initial problems before a developer ever sees the code.

> - Security and auditability: Maintain full control with audit logging and enterprise-grade access management…"

## 8. Limitations & Beginner-Friendly Caveats

These are important to surface on a beginner slide so audiences don't over-promise Agent HQ to their own teams:

1. **Agents make mistakes — by design, their output is reviewed.**
   > "Agents can still make mistakes. That's exactly why their output is designed to be reviewed, compared, and challenged, not blindly accepted."
   > — Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/)

2. **Tier gating.** Third-party agents (Claude, Codex) are currently **only on Copilot Pro+ and Copilot Enterprise**. Copilot Free / Pro / Business users do not have them yet. Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/) ("Copilot Pro+ and Copilot Enterprise users can now run multiple coding agents…"; "Access to Claude and Codex will soon expand to more Copilot subscription types").
3. **Each session = one premium request.** Running the same task across three agents in parallel costs three premium requests. Beginners should understand the billing implication. Source: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/) ("Each coding agent session consumes one premium request").
4. **Third-party agents must be explicitly enabled** in user settings, and enterprise admins can block them via AI Controls. Sources: [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/) ("Claude and Codex must be explicitly enabled in your settings before use"); [Agent management for enterprises — GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/enterprise-management) (AI Controls policy states).
5. **Parallelism has a trade-off.** GitHub itself warns that agent drafts can take minutes to an hour, and parallel agents in the same repo can create merge conflicts if they touch overlapping files. Source: [How to orchestrate agents using mission control — The GitHub Blog](https://github.blog/ai-and-ml/github-copilot/how-to-orchestrate-agents-using-mission-control/) ("your agents might spend a few minutes to an hour on a draft"; "Agents working in parallel can create merge conflicts if they touch the same files").
6. **Not a standalone product.** There is no separate "Agent HQ SKU" — it ships as features inside Copilot. Don't mislead audiences into thinking it's something you buy. Source: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/) ("You're accessing agents through your existing paid Copilot subscription").
7. **Partner agent list is not final.** Google / Cognition / xAI were announced but not shipped at the time of this research; availability and features may change. Sources: [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/) ("Over the coming months, coding agents from Anthropic, OpenAI, Google, Cognition, and xAI will be available…") and [Pick your agent — The GitHub Blog](https://github.blog/news-insights/company-news/pick-your-agent-use-claude-and-codex-on-agent-hq/) ("we're actively working with partners, including Google, Cognition, and xAI").
8. **Watch the name.** "Mission control" is sometimes used interchangeably with Agent HQ; strictly, Agent HQ is the platform/vision and mission control is one of its surfaces (see [Introducing Agent HQ — The GitHub Blog](https://github.blog/news-insights/company-news/welcome-home-agents/), where Agent HQ is the "vision" and mission control is the "single command center" within it).
